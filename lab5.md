# Функциональные требования к информационной системе библиотеки
## Основная функциональность системы
**1.Управление пользователями**

  * Регистрация и авторизация пользователей

  * Ведение профилей читателей

  * Управление правами доступа

  * Восстановление паролей

  * История действий пользователей

**2.Каталогизация фонда**

  * Ввод и редактирование библиографических записей

  * Классификация материалов

  * Штрихкодирование документов

  * Ведение инвентарных номеров

  * Учет местонахождения материалов

**3.Учёт и выдача литературы**

  * Оформление выдачи/возврата

  * Контроль сроков

  * Учет задолженностей

  * Формирование читательских билетов

  * История движения документов

**4.Администрирование**

  * Управление настройками системы

  * Мониторинг работы

  * Резервное копирование

  * Восстановление данных

  * Журналирование действий

**5.Отчётность**

  * Формирование стандартных отчетов

  * Аналитические отчеты

  * Статистика использования

  * Отчетность по задолженностям

  * Мониторинг работы библиотеки

## Функциональные требования
**Требования к управлению пользователями:**

FR1: Система должна обеспечивать регистрацию новых читателей с заполнением обязательных полей

FR2: Система должна позволять изменять персональные данные пользователя

FR3: Система должна контролировать права доступа в зависимости от роли пользователя

FR4: Система должна вести журнал действий пользователей

**Требования к каталогу:**

FR5: Система должна обеспечивать ввод библиографических данных по установленным стандартам

FR6: Система должна поддерживать многокритериальный поиск

FR7: Система должна хранить историю изменений записей

FR8: Система должна обеспечивать штрихкодирование документов

**Требования к учету литературы:**

FR9: Система должна фиксировать факт выдачи/возврата документа

FR10: Система должна контролировать сроки возврата

FR11: Система должна уведомлять о просроченных документах

FR12: Система должна вести учет задолженностей

**Требования к отчетности:**

FR13: Система должна формировать отчеты по движению фонда

FR14: Система должна генерировать статистику использования

FR15: Система должна предоставлять аналитические отчеты по запросу

FR16: Система должна сохранять шаблоны отчетов

**Требования к администрированию:**

FR17: Система должна обеспечивать резервное копирование данных

FR18: Система должна иметь механизмы восстановления

FR19: Система должна контролировать целостность данных

FR20: Система должна поддерживать обновление программного обеспечения

**Высокий приоритет:**

FR13-FR16 (отчетность)

**Средний приоритет:**
